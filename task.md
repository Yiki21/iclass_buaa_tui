# 可用性与体验改进任务

状态标记：
- `[x]` 已完成并通过本地验证
- `[~]` 已部分完成，仍有后续增强
- `[ ]` 未开始

## P0：登录与故障恢复

### 1. 登录诊断面板 `[x]`
- 目标：登录失败时用户能直接判断是网络、账号密码、验证码、SSO 页面变化还是 iClass 接口异常。
- 改动：
  - [x] 在 TUI 增加“查看详情”弹窗，展示错误链、最终 URL、HTTP 状态、页面线索、建议动作。
  - [x] CLI 增加 `--debug-login`，输出同样的结构化诊断。
  - [x] 对敏感字段做脱敏，禁止输出密码、Cookie、token。
- 验收：
  - [x] DNS/TLS/超时/HTTP 非 200/验证码/账号错误分别有可区分提示。
  - [x] 错误信息能直接复制到 issue，且不包含密码。
  - 验证：`cargo check`、`cargo test`。

### 2. VPN 连通性自检 `[x]`
- 目标：用户登录前就知道当前网络是否能访问 WebVPN、SSO、iClass、BYKC。
- 改动：
  - [x] 新增 CLI：`iclass_buaa_tui doctor`。
  - [x] TUI 登录页增加 `D` 自检入口。
  - [x] 检查 DNS、WebVPN 首页、SSO 登录页、iClass login API、BYKC CAS 入口。
- 验收：
  - [x] 网络问题不会再表现为模糊的“登录失败”。
  - [x] 输出包含每个检查项的耗时、状态和下一步建议。
  - 验证：`cargo check`、`cargo test`。

### 3. 验证码支持 `[~]`
- 目标：SSO 触发验证码时不要求用户手动打开浏览器。
- 改动：
  - [x] 解析 CAS 页面验证码配置。
  - [x] TUI 显示外部临时图片路径，允许输入验证码后继续提交同一 session。
  - [x] CLI 遇到验证码时给出明确失败原因和浏览器替代方案。
  - [ ] 可选增强：在终端内直接渲染验证码图片，减少用户打开临时文件的步骤。
- 验收：
  - [x] 触发验证码时不会误报账号密码错误。
  - [~] TUI 内可完成一次验证码登录；仍建议用真实触发验证码的账号做一次手工回归。
  - 验证：`cargo check`、`cargo test`。

## P1：核心使用体验

### 4. 首次配置向导 `[ ]`
- 目标：降低首次使用门槛。
- 改动：
  - TUI 首次启动检测无配置时进入向导。
  - 支持选择直连/VPN、保存账号、测试登录、生成配置文件。
  - 对配置文件权限自动修正或给出命令。
- 验收：
  - 新用户不读 README 也能完成配置。
  - `vpn_password` 配置权限不合规时有明确修复路径。

### 5. 状态栏升级为事件日志 `[x]`
- 目标：避免长错误或多阶段操作被一行状态栏截断。
- 改动：
  - [x] 底部状态区升级为事件区，保留最近 8 条事件。
  - [x] 支持 `e` 打开日志弹窗，`y` 复制最近错误，`C` 清空日志。
  - [x] 区分 info/warn/error/success 样式。
- 验收：
  - [x] 登录、刷新、签到、BYKC 操作的阶段性结果都可追溯。
  - [x] 长错误不会挤坏布局。
  - 验证：`cargo test`。

### 6. 登录后后台预热 `[ ]`
- 目标：减少首次切换 BYKC 或打开详情时的等待。
- 改动：
  - VPN 登录成功后异步预热 BYKC token、课程列表和统计。
  - iClass 课程刷新和 BYKC 刷新互不阻塞。
  - UI 显示每个模块的独立加载状态。
- 验收：
  - 登录后切换 BYKC 不需要重新等待完整登录流程。
  - 任一模块失败不影响另一个模块可用。

## P1：自动化可靠性

### 7. 自动签到 dry-run 解释器 `[ ]`
- 目标：用户知道为什么某门课会签/不会签。
- 改动：
  - `plan --dry-run` 输出每个目标的命中规则、排除规则、时间窗口、跳过原因。
  - TUI 增加“今日自动化预览”页。
- 验收：
  - include/exclude 配错时可以从输出中直接看出。
  - 已签、未到时间、过期、缺少 courseSchedId 都有明确说明。

### 8. 调度器健康检查 `[x]`
- 目标：安装自动签到后能确认它真的在运行。
- 改动：
  - [x] `install-autologin` 结束后提示可执行的健康检查命令。
  - [x] 新增 `autologin-status` 命令。
  - [x] Linux/macOS/Windows 分别展示调度器状态和最近日志位置/查看路径。
- 验收：
  - [x] 用户能一条命令确认自动签到是否启用、下次何时触发、上次是否失败。
  - 验证：`cargo check`、`cargo test`。

### 9. 失败重试与退避策略 `[ ]`
- 目标：降低临时网络抖动导致的失败率。
- 改动：
  - 区分可重试错误和不可重试错误。
  - 网络/5xx/超时使用指数退避；账号密码/验证码/配置错误直接失败。
  - CLI 输出每次重试的原因和下一次等待时间。
- 验收：
  - 短暂网络故障不立刻导致自动签到失败。
  - 凭据错误不会浪费重试次数。

## P2：界面效率

### 10. 课程搜索与过滤 `[ ]`
- 目标：课程多时快速定位。
- 改动：
  - iClass 和 BYKC 列表支持 `/` 搜索。
  - 支持按今天/本周/可签到/已签到筛选。
  - 搜索状态显示在列表标题。
- 验收：
  - 100+ 条课程内能快速定位目标。
  - 搜索不改变原始数据，只影响显示。

### 11. 操作确认与撤销 `[ ]`
- 目标：避免误点报名、退选、签退。
- 改动：
  - BYKC 报名/退选/签退前显示确认弹窗。
  - 对不可逆操作提示课程名、时间窗口、地点状态。
  - 最近一次可撤销操作若上游支持则提供快捷入口。
- 验收：
  - 不会一键误退选。
  - 确认弹窗信息足够用户判断。

### 12. 外部二维码体验优化 `[ ]`
- 目标：二维码签到更稳定。
- 改动：
  - 外部二维码页面显示课程名、刷新状态、最后更新时间。
  - 失败时页面显示错误而不是静默停留旧二维码。
  - TUI 显示外部二维码文件路径和刷新状态。
- 验收：
  - 浏览器页面能判断二维码是否仍在刷新。
  - 课程切换后旧二维码自动失效或明确提示。
